CSS

HTML används främst för att ge sidan sitt innehåll och struktur CSS kommer sedan in för att ge webbsidan sin färg och form. Här kommer vi visa lite på hur vi strukturerar vår CSS kod samt hur koden förvandlar ett annars tråkigt html dokument till något lite roligare.

Selektorer

Inline

CSS går att skriva direkt i html dokumentet antingen under < style > taggen eller inom den taggen du arbetar med genom så kallad inline CSS. Vi använder oss utav ett separat dokument som vi länkar till i vår head element. Detta underlättar då vi kommer använda samma CSS kod för att styla alla våra html sidor.

Först och främst måste vi se till att vi lägger till css på rätt HTML element. Man kan då använda sig utav inline css men för större projekt är det rekommenderat att man istället kallar på elementen inuti ett CSS dokument.

För att kalla på ett element i css behöver man bara skriva in vilket element du vill justera och därefter ett par klamrar därefter som du deklarerar din ändringar.


                            <head>
                                <meta charset="UTF-8">
                                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                <meta http-equiv="X-UA-Compatible" content="ie=edge">
                                <title>Exampel</title>
                            </head>
                            <style> </style

ID och class

Säg då att du inte vill ändra på alla din paragrafer (< p>) när du skriver din CSS utan vill endast ändra på en rad text. För att göra detta kan vi använda oss utav ID och class attribut. Med dessa kan man lätt kalla på individuella element i din HTML kod. För enstaka element rekommenderas det att du använder dig utav ID och för en grupp element rekommenderas det att du använder utav class. För att ge ett HTML element ett ID eller en klass skriver du bara in class=”[Namn]” eller id=”[Namn]” inom elementets klamrar när du skapar den.

När du sedan vill kalla på ett element med en ID eller class så skriver du in klassen eller id:ts namn med ett # framför id:t och en punkt (.) framför klasser och ett par klamrar efter det.


                            <head>
                                <meta charset="UTF-8">
                                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                <meta http-equiv="X-UA-Compatible" content="ie=edge">
                                <title>Exampel</title>
                            </head>
                            <style> 
                            <.classEX{}
                            <#idEX{}
                            </style
                            
                            <div class="classEX">
                                <div id="idEX">
                                </div>
                            </div>

Styling

Det finns flera olika värden man kan använda för att justera olika element i HTML med hjälp av CSS. Majoriteten av dessa har inte så jättestor betydelse på hela hemsidan förutom utseende så vi kommer snabbt gå igenom några attribut som användes för att bygga denna hemsida.


                        .classEX{
                            Background-color:black;
                            font-family:"comic sans";
                            width: 50%;
                            height: 50%;
                            border: 1px;

                        }

Boxmodellen

Den så kallade boxmodellen är term som används när man pratar om en sidas layout och design. Vad det innebär är att alla HTML element ligger i en “box” som täcker hela elementet. Vårt element täcks först av padding, därefter en border som går runt elementet och paddingen och därefter margin som ser till att det finns tom plats run allting.

Position

Även om det låter som det så bestämmer inte position vart element ska ligga på en hemsida. Istället så bestämmer position hur ett element anpassar sig efter HTML:ens flow. Flow innebär vart någonstans i koden ett element ligger. Som exempel kan vi använda oss utav denna sidas header. Headern har en sticky position vilket innebär att oavsett hur långt ner du scrollar kommer headern stanna kvar på sidans topp.


                        header{
                        display: grid;
                        position: -webkit-sticky;
                        position: sticky;
                        top: 0;
                        font-family: 'Ubuntu Condensed', sans-serif;
                        }

Positionering

Det finns flera olika sätt att bestämma vart någonstans ett element ska ligga på din hemsida.

Float

Float är ett äldre attribut som man använder sig av för att positionera ett element utanför dokumentets flow. Om ett element har attributet float: left så kommer elementet då att flytta sig till hemsidans vänstra sida och flytta på alla andra element så att det får plats just där.

Flexbox

Flexbox är ett annat lite modernare sätt att positionera ett element. Just denna metod använder sig av boxmodellen och där alla element radas upp efter varandra i en designerad flex-container.

Grid

Sist men inte minst så har vi grid. Man kan beskriva denna metod som att placera dina element i ett rutnät. Man börjar först att skapa en grid-container. Alla element som skapas innanför denna container kommer att omvandlas till ett grid-element som placeras inom containerns rutnät i den ordning som de skapades i. För att sedan bestämma vart grid-elementen ska ligga ändrar vi på mellanrummet mellan elementen. Det horisontella mellanrummet kallas för grid-row medans det vertikala kallas för grid-column.

Genom att använda sig utav attributen grid-column-gap och grid-row-gap kan vi bestämma hur mycket mellanrum det ska vara mellan de olika elementen i rutnätet. Man kan också bestämma hur mycket plats ett grid element kan ta upp i rutnätet genom grid-column/row-start och grid-column/row-end.

Om vi har vill att en av våra element ska ta mer plats än de andra i en grid container kan vi sätta elementets början genom grid-column/row-start och därefter hur mycket plats det elementet ska ta upp genom grid-column/row-end.


                            header{
                                display: grid;
                                position: -webkit-sticky;
                                position: sticky;
                                top: 0;
                                font-family: 'Ubuntu Condensed', sans-serif;
                                }
                            .ikon{
                                grid-column-start: 1;
                                }
                            .navbar ul{
                                padding: 0;
                                margin: 0;
                                grid-column: 3 / span 2;
                                }
                            .searchField{
                                grid-column-start: 5;
                                }
